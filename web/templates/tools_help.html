1.1 Connect to WLAN

Connect the device to the internal WLAN "Tarkin" using the password provided in Syspass.

1.2 Adjust WLAN settings

Most modern devices do have some settings like mac randomization which should be disabled. Please check following points to prepare the device!

  • Disable Proxy Services
    Features like Apple Private Relay or VPNs must be turned off.

  • Disable MAC Address Randomization
    Turn off “Private Wi-Fi Address” on iOS. Or "Mac-Addresstype" on Android to Phone-MAC

1.3 Add the Device

After the device has successfully joined the WLAN and all recommended settings have been applied, navigate to Manage Devices:

  • Evidence ID: A000000001
  • Description: Apple iPhone 16 Pro
  • Examiner: stpg83

1.4 Assign the Lease

Note: Leases cannot be revoked yet!

  • Find the correct IP/MAC from the device.
  • Select the corresponding lease entry and assign it.

2.1 View: Blocked

  • View recent blocked connections with IP, port, protocol, ISP info.
  • Status icons show if the IP is enriched or unknown.
  • Use ISP filters to allow or disallow traffic related to the ISP.
  • Each shows (active / total) firewall rule count.
  • Use “Update Firewall” to sync rules per ISP.

2.2 View: Passed

  • View recent passed connections with IP, port, protocol, ISP info.
  • Use device filters to narrow down.
  • Status icons show if the IP is enriched or unknown.

2.3 View: ISP

  • See which ISPs the device communicated with.
  • You have the ability to add or remove individual firewall rules for each device.

2.4 View: DNS

  • See which dns names resolved to a specific IP address.
  • Status Icons in the title bar indicate whether the IP is currently blocked or allowed
  • Search Bar at the top enables search by IP address or domain (e.g., apple.com)
  • based on parsed data from live traffic and shown with timestamp, query type, domain, source IP, and resolved destination IP.
  • You have the ability to add or remove individual firewall rules for each device.

2.5 View: Firewall Rules

  • Displays all active firewall rules for the selected device.
  • Each rule shows action, protocol, source IP, destination IP, port, and ISP.
  • Indicates whether the rule was added manually or automatically.
  • “DNS” column shows related domain name, if available.
  • Use the “Remove” button to delete individual rules.
  • If no rules are visible, check if the device is selected or rules exist.

2.6 View: Domain to ISP Lookup

  • Enter a domain name to see which IPs and ISPs are associated with it.
  • Helps determine which ISPs host or serve a specific domain.
  • Useful for evaluating if a domain can be allowed or blocked based on its ISP.
  • Results are shown in a simple table with IP and corresponding ISP.
